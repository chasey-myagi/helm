# PRODUCT.md — helm

## Register
product

## Product purpose
helm 是一个**单任务实时工作看板**。一个 coding agent(Claude Code / Codex)在做某件长任务时,
用一个 `dash` CLI 加一个**短命的本地进程**,把"它现在在干嘛 + 工作上下文"实时渲染成一个网页,给
**开着这个网页的人**看。人还能在网页里改"目标 + 操舵指令",agent 在下一个检查点读到后跟随调整。

它按优先级回答:**它现在在干嘛 → 基于什么计划/决策/假设 → 需不需要我介入 → 干到哪了**。
不是事后汇报,是给"在场的操作者"实时看的工作台。

## Users
- 单一用户:一个独立开发者本人,技术背景强,常同时编排多个 coding agent。
- 场景:人在另一个窗口(通常是深色 IDE)里干别的活,余光偶尔扫这块看板;被"需要拍板/卡住"叫回来处理;
  偶尔编辑目标来给跑偏的 agent 纠偏。母语中文。
- 不是给团队或客户的汇报面,屏幕上同时只盯一个任务。

## Content model(grilling 逐条锁定)
**左栏 — 常驻,永不随滚动离开视野:**
- **Now**:当前动作一句话 + 状态(running / blocked / waiting / done)+ 细进度。
- **计划**:有序步骤,每步带状态;"进行中"那步就是 Now。计划即进度,二合一,不另设勾选清单。
- **操舵区(人类可编辑、回写磁盘)**:目标 + 一段自由操舵指令(如"别动 schema""先跑 staging""放弃方案 A")。
- **需要你拍板**:只放真实的 HITL 阻塞问题,绝不凑数。
**右栏 — 流水,滚动:**
- **活动时间线**:事件流,最新在上;踩的坑/死路作为 warn/fail 事件落在这里。
- **决策 & 假设**:agent 当前据以行动的关键决策和假设,标出"未经确认的假设",便于早纠偏。
- **产物**:链接 + 图片 + 数据。

## Behavior
- blocked 或新增"需要你拍板" → 浏览器通知 + tab 标题加 🔴 计数(人在别的窗口也能被叫回)。
- 进程:任务结束(`dash done`)/ tab 关闭心跳超时 / 空闲超时,任一即自动退出。**绝不常驻、绝不写 restart、绝不开机自启**(用户被会自动复活吃内存的本地栈坑过)。
- 实时:服务端 SSE 推送,无轮询、无闪烁、无整页 reload。
- agent 侧只调 `dash`,**绝不手写 HTML**(每次更新只花几个 token)。

## Tone
克制、专业、信息层次清楚。像一份精致的工作台 / 仪表,不是营销页,不是花哨监控大屏。

## Anti-references(不要变成这些)
- 不要 SaaS 的 hero-metric 模板(超大数字 + 渐变 + 一排小 stat)。
- 不要千篇一律等大卡片网格、图标+标题+说明无限重复。
- 不要"AI 一眼假"的通用**深蓝监控大屏** reflex(这是本品类的第一反射,要主动避开)。
- 不要渐变文字、不要彩色侧边竖条(side-stripe)、不要无意义玻璃拟态。

## Strategic principles
- **焦点优先**:"现在在干嘛 + 要不要我管"必须一眼定位,且永不滚走。
- **诚实**:如实呈现 agent 只能在检查点跟随人类操舵,不假装能中途打断。
- **安静但叫得醒**:平时不打扰,真卡住时主动喊人。
